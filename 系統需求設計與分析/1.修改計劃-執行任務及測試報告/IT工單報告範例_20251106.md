## 📊 報告結構說明

### 1. 工單基本資訊
- **工單編號**：格式為 `AETIM-{威脅ID}-{時間戳記}`
- **優先級**：根據風險分數自動設定
  - `P0`：風險分數 >= 9.0（嚴重威脅）
  - `P1`：風險分數 < 9.0（高風險威脅）
- **標題**：簡要描述威脅和受影響資產
- **建立時間**：ISO 8601 格式的時間戳記

### 2. 受影響資產
- **主機名稱**：受威脅影響的主機名稱
- **IP 地址**：主機的 IP 地址
- **負責人**：資產負責人

### 3. 威脅資訊
- **CVE ID**：漏洞識別碼
- **威脅標題**：完整的威脅描述
- **風險分數**：系統計算的風險分數（0-10）
- **威脅來源**：威脅情報來源（CISA_KEV、NVD、RSS 等）
- **CVSS 分數**：CVSS 評分（如果有）

### 4. 修補建議
根據威脅類型和資產特性自動生成的修補建議，包括：
- CVE 詳細資訊連結
- 相關安全公告連結
- 廠商特定建議
- 臨時緩解措施建議

## 📁 報告檔案位置

報告檔案儲存在以下目錄結構中：
```
reports/
└── 2025/
    └── 202511/
        ├── it_ticket_20251106_083704.text  (文字格式)
        └── it_ticket_20251106_083704.json  (JSON 格式)
```

### 檔案命名規則
- **格式**：`it_ticket_YYYYMMDD_HHMMSS.{format}`
- **範例**：`it_ticket_20251106_083704.text`
- **說明**：
  - `it_ticket`：報告類型
  - `20251106`：日期（YYYYMMDD）
  - `083704`：時間（HHMMSS）
  - `.text` 或 `.json`：檔案格式

## 📝 報告生成條件

IT 工單會在以下條件下生成：

1. **風險分數閾值**：風險分數 >= 7.0（可在 `config.yaml` 中設定）
2. **資料庫中有高風險威脅**：`T_Validated_Threats` 表中有符合條件的記錄
3. **手動觸發**：透過 Web 界面手動生成
4. **自動觸發**：當關聯分析發現高風險威脅時自動生成

## 🎯 使用場景

### 場景 1：手動生成 IT 工單
1. 訪問 Web 界面：`http://localhost:5001`
2. 在「報告生成」區域選擇「IT 工單」
3. 選擇格式（Text 或 JSON）
4. 點擊「生成報告」按鈕
5. 等待報告生成完成
6. 查看報告檔案路徑

### 場景 2：自動生成 IT 工單
- 當關聯分析引擎發現高風險威脅（風險分數 >= 7.0）時
- 系統會自動生成 IT 工單
- 工單會透過 Email 發送給 IT 團隊（如果已設定）

## 📌 注意事項

1. **資料庫狀態**：確保資料庫中已有高風險威脅資料
   - 如果沒有，請先執行收集任務和關聯分析

2. **報告格式**：
   - **Text 格式**：適合人類閱讀，結構化文字
   - **JSON 格式**：適合系統整合，結構化資料

3. **檔案儲存**：
   - 報告檔案會自動儲存到 `reports/yyyy/yyyymm/` 目錄
   - 檔案不會自動刪除，需要手動清理舊報告

4. **報告內容**：
   - 報告內容基於資料庫中的最新高風險威脅
   - 如果有多個高風險威脅，會選擇風險分數最高的
